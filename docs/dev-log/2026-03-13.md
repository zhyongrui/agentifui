Useful continuity notes for future sessions:

- runtime package exports use `dist`, while TypeScript types come from `src`
  - after adding new runtime value exports to `packages/shared` or `packages/db`, `npm run type-check` can still pass while runtime values are `undefined`
  - the minimal fix is to rebuild the affected workspace packages before route tests or ad-hoc runtime smoke:
    - `npm run build --workspace @agentifui/shared`
    - `npm run build --workspace @agentifui/db`
- the new upload route is intentionally conversation-scoped:
  - `POST /workspace/conversations/:conversationId/uploads`
  - this keeps attachment authorization simple for Phase 1 because file ownership and replay both ride on the existing conversation boundary
- the current upload implementation is JSON + base64, not multipart
  - this was chosen to avoid adding a new Fastify multipart layer while keeping the `/api/gateway/*` proxy unchanged
  - if uploads need to grow beyond the current Phase 1 limit, `R13-B` or later should revisit multipart/object storage presign flows
- the Playwright attachment regression uses the stable DOM hook `#chat-attachment`
  - it uploads `Buffer.from('Policy attachment')`
  - it waits for `/api/gateway/workspace/conversations/:id/uploads`
  - it expects the transcript chip and replay section to both surface `brief.txt`
- public access/browser testing context from the prior round still applies
  - temporary `cloudflared tunnel --url http://127.0.0.1:3112 --no-autoupdate`
  - stable `80/443` reverse-proxy hardening remains deferred to `R13-B`
