## Continuity Notes

- comment creation is intentionally conversation-scoped
  - the single route is `POST /workspace/conversations/:conversationId/comments`
  - `targetType` determines whether the note attaches to a message, run, or artifact
- comments are stored in `workspace_comments`
  - they are not embedded back into `conversations.inputs` or `runs.outputs`
  - this avoids losing notes when transcript or run payloads are regenerated
- owner surfaces can create comments today
  - shared surfaces will read the same comments once they render the augmented conversation/artifact payloads
  - shared write permissions are still a later collaboration policy task
