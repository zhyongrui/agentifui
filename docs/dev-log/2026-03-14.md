## P2-A5 Conversation Organization

Completed `P2-A5` by turning conversation history management into a real persisted capability across both `/chat` and `/chat/[conversationId]`.

Implemented in this round:

- shared contracts now include:
  - `WorkspaceConversation.pinned`
  - `WorkspaceConversationListItem.pinned`
  - `WorkspaceConversationUpdateRequest` / `WorkspaceConversationUpdateResponse`
- gateway and services now include:
  - `PUT /workspace/conversations/:conversationId`
  - in-memory and persistent workspace service support for `title`, `status`, and `pinned` updates
  - deleted conversations and their runs are filtered out of normal workspace reads
  - history ordering is now `pinned desc, updatedAt desc`
  - audit writes for `workspace.conversation.updated` and `workspace.conversation.deleted`
- web history/detail surfaces now include:
  - `/chat` actions for pin, rename, archive, restore, and delete
  - `/chat/[conversationId]` management panel for the same actions
  - archived conversations stay readable but disable composer, attachments, retry, regenerate, and suggested prompts
  - deleted conversations redirect back to `/chat?deleted=1`
- test coverage now includes:
  - unit and route coverage for conversation updates
  - persistence coverage across restart boundaries
  - browser coverage for `pin -> rename -> archive -> restore -> delete`

Useful continuity notes for future sessions:

- use `npm run test:e2e` on this host, not raw `npx playwright test`
  - the wrapper starts the expected project stack
  - direct Playwright launches can fail here with missing browser runtime libraries such as `libatk-1.0.so.0`
- Playwright response waits needed to be widened from `60s` to `120s`
  - shared Postgres load occasionally pushes auth requests past one minute
  - the helper lives in `tests/e2e/phase1-flows.spec.ts`
- the conversation-organization browser flow should seed a conversation directly in Postgres
  - this avoids consuming launch quota when the test only needs a persisted conversation shell
  - the helper is `seedWorkspaceConversation(email)` in `tests/e2e/phase1-flows.spec.ts`
- after `Restore`, do not assert the `Send message` button is enabled
  - the composer submit stays disabled while the draft is empty
  - assert the `Message` textarea and `Attachments` input are enabled instead
- stop-flow browser assertions should accept either `stopped` or `succeeded`
  - the stop request is still exercised
  - on fast completions, the run can naturally finish before the stop request wins the race
- earlier public-access/browser-testing context still applies
  - temporary external access can be provided with `cloudflared tunnel --url http://127.0.0.1:3112 --no-autoupdate`
  - long-lived `80/443` ingress remains the stable path for formal manual QA
