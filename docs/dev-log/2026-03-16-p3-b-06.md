## Continuity Notes

- `workspace.pending_action.responded` remains the generic HITL audit record
  - use the new `workspace.tool_execution.*` actions when you need tool-specific analytics or filtering
- the current event split is:
  - `approval_requested` when a tool-approval HITL step is emitted
  - `approval_decided` when an operator responds
  - `completed` for approved/immediate successful tool runs
  - `blocked` for rejected or cancelled approval-gated tool runs
  - `failed` reserved for runtime tool failures
- persisted audit verification is easiest with:
  - `action=workspace.tool_execution.completed`
  - `traceId=<chat trace id>`
  - both together keep the returned audit window small and deterministic
