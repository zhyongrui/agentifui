## Live Session Runtime Fix Update

After exposing the app through the live preview URL and logging in through the real UI, another frontend-only runtime issue surfaced on `/apps`.

Observed failure:

- after successful login, `/apps` crashed with:
  - `Functions are not valid as a child of Client Components`

Root cause:

- `apps/web/src/app/(main)/apps/page.tsx`
- `apps/web/src/app/(main)/settings/profile/page.tsx`
- `apps/web/src/app/(main)/settings/security/page.tsx`

These pages were server components rendering a client component (`ProtectedPage`) with a render-function child. That crosses the Server/Client boundary incorrectly in Next.js and only fails once the real page is rendered in the browser flow.

Fix applied:

- added `apps/web/src/lib/use-protected-session.ts`
  - moved session-read + redirect logic into a client hook
- converted protected route pages into explicit client components
- removed `apps/web/src/components/protected-page.tsx`
  - eliminated the render-prop pattern that caused the crash
