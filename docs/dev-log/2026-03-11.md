## Package Management Decision

The original design direction referenced `pnpm`, but local execution repeatedly stalled on the available `pnpm` setup in this environment.

To keep delivery moving, the implementation repository was switched to:

- `npm@10.9.2`
- root `workspaces`
- generated `package-lock.json`

This is an implementation-repository decision, not a product-spec decision. It should be revisited later if the team wants to standardize back to pnpm.
