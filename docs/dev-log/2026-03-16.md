Useful continuity notes for future sessions:

- chunking is intentionally heuristic-first for now
  - markdown sources always use `markdown_sections`
  - `url` and `file` sources use `paragraph_windows` unless the content visibly contains markdown headings
  - retrieval ranking and semantic search are not implemented yet; this round only prepares source material for those later slices
- `knowledge_source_chunks` currently stores raw chunk text and preview text
  - later retrieval work should read from this table instead of reparsing `knowledge_sources.source_content`
  - if a future migration changes chunk shape, remember to update:
    - `replaceSourceChunks(...)`
    - backup allowlist in `scripts/lib/backup-helpers.mjs`
- `P3-A-03` does not add any new browser coverage
  - browser source-management coverage remains a later task in `P3-A-10`
  - persistence-heavy ingestion transition coverage remains a later task in `P3-A-09`
