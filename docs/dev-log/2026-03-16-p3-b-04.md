## Continuity Notes

- `WorkspaceRun.toolExecutions` is now the stable read model for tool-stage replay
  - keep writing the legacy `outputs.toolCalls` / `outputs.toolResults` keys for now
  - read paths should prefer `toolExecutions` and only derive from legacy keys as fallback
- legacy fallback currently maps one execution per `toolCall`
  - `attempt` defaults to array index order
  - `latencyMs` is `null` if old payloads did not store it
  - fallback timestamps use `finished_at` or `created_at` from the run row so replay remains stable across reads
- `chat.ts` now records tool latency at the moment the runtime returns
  - this intentionally excludes downstream SSE text streaming time
  - if future runtimes expose real per-tool timings, replace the current equal-split placeholder in `buildToolExecutions()`
- root `npm run type-check` no longer relies on `turbo` to run the web package
  - it now runs non-web packages through `turbo`
  - then runs `@agentifui/web` type-check serially
  - keep this until the Next 16 `.next/types/routes.ts` flake is gone on this host
