## Continuity Notes

- tool registry persistence now lives in `workspace_app_tool_overrides`
  - the unique key is `(tenant_id, app_id, tool_name)`
  - backup/export allowlists must continue to include this table
- admin app summaries still originate from the existing admin service
  - route handlers enrich those summaries with live tool registry state before sending the API response
  - if a future change edits `AdminAppSummary`, keep the enrichment layer in sync
- chat completion validation now rejects disabled tool names before runtime execution
  - request `tools`
  - function `tool_choice`
  - assistant `message.tool_calls`
  - the next slice `P3-B-03` should build on this contract instead of bypassing it
- `npm run test:unit` must explicitly exclude all `*persistence.test.ts` files
  - leaving them inside unit causes shared-Postgres DDL conflicts on this host
- `npm run test:persistence` now runs the three persistence files sequentially
  - do not parallelize them on the shared local database
- browser assertions on this host need generous waits for:
  - `/admin/*` headings
  - replay/run-history panels
  - retrieval-backed citation/source rendering
  - default `5s` `toBeVisible()` checks are too brittle for the slowest runs
