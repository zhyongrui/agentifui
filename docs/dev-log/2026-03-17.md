## P3-C-06 Shared commenter/editor permission modes

Completed `P3-C-06` by extending shared conversation access from `read_only` to:

- `read_only`
- `commenter`
- `editor`

Shipped surfaces:

- owner share creation now accepts `access` and persists it on `workspace_conversation_shares`
- shared collaborators can post comments through `POST /workspace/shares/:shareId/comments`
- shared editors can update title / pinned / archived through `PUT /workspace/shares/:shareId/conversation`
- shared transcript now shows access badges, message comment composers, and an editor-only metadata panel
- shared artifact preview now inherits `shareAccess` and enables comments for commenter/editor shares
- `/chat/[conversationId]` share panel now lets the owner choose the access mode when creating or reactivating a share
