## Continuity Notes

- `run.failure` and `toolExecution.failure` are now related but distinct
  - run-level failure still means the run itself failed
  - tool-level failure now captures failed attempts inside otherwise-succeeded runs
- persisted rows can still lack `toolExecution.failure`
  - readers now derive it from legacy `metadata.failureReason` plus result content
  - new writes should always include the structured `failure` payload
- current tool-stage taxonomy codes are:
  - `tool_timeout`
  - `tool_provider_error`
  - `tool_approval_rejected`
  - `tool_approval_cancelled`
  - `tool_approval_expired`
- current tool-stage taxonomy stages are:
  - `tool_execution`
  - `tool_approval`
